# Security Policy

## Supported Versions

Fast4d does not currently follow semantic versioning across releases. Security
fixes are applied only to the latest code on the `main` branch.

| Version         | Supported          |
| --------------- | ------------------- |
| `main` (latest) | :white_check_mark: |
| Older commits   | :x:                 |

## Reporting a Vulnerability

If you discover a security vulnerability in Fast4d, please report it privately
rather than opening a public issue.

- **Contact:** jtapiaca@asu.edu
- **Response time:** you can expect an initial response within 5 business days.
- **What to include:** a description of the vulnerability, steps to reproduce
  it, and its potential impact.
- **What to expect:** if the report is accepted, a fix will be prioritized and
  you'll be credited (unless you prefer to stay anonymous) once it ships. If
  declined, you'll receive an explanation of why it's out of scope.

Please do not disclose the issue publicly until it has been addressed.
